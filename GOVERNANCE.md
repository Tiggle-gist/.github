# Governance

Tiggle 팀의 의사결정 방식과 운영 규칙을 정의합니다.

---

## 팀 구성

| 역할 | 담당 | 도메인 |
|------|------|--------|
| 팀장 | 김예원 | Software / AI |
| 팀원 | 강지훈 | Software / AI, Communication |
| 팀원 | 유수진 | On-Device / 회로 |
| 팀원 | 미정 | Hardware / Structure |

---

## 의사결정 방식

```
이슈 발생
    ↓
Slack에서 의견 수렴
    ↓
합의되면 → 즉시 결정
합의 안 되면 → 정기 회의에서 논의
    ↓
그래도 안 되면 → 팀장 최종 결정
```

**도메인별 결정권:**
각 도메인의 기술적 결정은 해당 담당자가 사실상 결정권을 가진다.
도메인 간 충돌이 생길 때만 팀장이 개입한다.

---

## 회의

- **주기:** 주 1회 정기 회의
- **시간:** 추후 확정
- **형식:**
  - 지난주 Action Item 확인
  - 이슈 논의 및 결정
  - 다음주 Action Item 정리

- **규칙:**
  - 안건은 회의 전날까지 Slack `#회의안건`에 공유
  - 회의록은 당일 `notion`에 업로드
